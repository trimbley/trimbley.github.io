瀏覽器指紋 / User-Agent 線上檢視器

每個瀏覽器都會向網站暴露一組訊號：User-Agent、canvas 與 WebGL 渲染、字型、語言、時區、螢幕尺寸——它們組合起來，往往不需要任何 cookie 就能識別出你。本工具在你的瀏覽器本地讀取這些訊號並展示給你；不上傳任何資料，掃描全程在你的裝置上完成。

你此刻的瀏覽器指紋

實時讀取自你的瀏覽器。所有數值只顯示在本頁面，不會傳送到任何伺服器。

什麼是瀏覽器指紋？

瀏覽器指紋是用你瀏覽器對外暴露的屬性拼出的一份畫像：User-Agent 字串、已裝字型、螢幕解析度、時區、語言列表，以及裝置透過 canvas 和 WebGL 渲染圖形的方式。由於這些屬性的組合常常是獨一無二的，即使你遮蔽 cookie 或用無痕模式，網站仍可能在你再次到訪時認出你。

canvas 與 WebGL 指紋的原理

當頁面把隱藏的文字和圖形畫到 canvas 再把畫素讀回時，你的 GPU、顯示卡驅動和字型渲染上的細微差異會產生一個穩定、且專屬於你這臺機器的影象雜湊。WebGL 指紋則以同樣方式讀取未被遮蔽的 GPU 廠商與渲染器。這兩個訊號之所以強，是因為它們能挺過大多數清除 cookie 的操作，並跨會話持續存在。

哪些訊號暴露得最多

研究者用「熵的位元數」衡量識別力：位元越多，和你取值相同的人越少。已發表研究把一份完整指紋估為約 18 位元，其中字型、User-Agent、canvas 和 WebGL 渲染器是最強的貢獻項。下圖展示了主要來源的相對權重。

FAQ

我的資料會被髮送到哪裡嗎？

不會。掃描完全在你的瀏覽器內用標準 Web API 完成；數值只顯示在頁面上，絕不上傳。

User-Agent 字串是什麼？

它是瀏覽器隨每個請求傳送的一行文字，標明瀏覽器名稱、版本和作業系統，是最古老也最基礎的指紋訊號之一。

canvas 雜湊為什麼重要？

把影象畫到 canvas 再讀回，會暴露 GPU 與字型渲染上的細微差異，產生一個常常專屬於你裝置、且跨會話穩定的雜湊。

清除 cookie 能隱藏指紋嗎？

不能。指紋來自裝置與瀏覽器屬性，而非儲存的 cookie，所以它能在你清除 cookie 或使用無痕模式後依然存在。

如何減少我的指紋？

使用會歸一訊號的瀏覽器（如 Tor 瀏覽器或 Brave），保持瀏覽器更新，並避免使用讓你顯得與眾不同的稀有字型、擴充套件和螢幕設定。

這些數值準確嗎？

它們直接讀取自你的瀏覽器，因此就是任何網站能看到的內容。部分屬性（deviceMemory、vendor、platform）已廢棄或為近似值，可能缺失。

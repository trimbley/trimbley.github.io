浏览器指纹 / User-Agent 在线查看器

每个浏览器都会向网站暴露一组信号：User-Agent、canvas 与 WebGL 渲染、字体、语言、时区、屏幕尺寸——它们组合起来，往往不需要任何 cookie 就能识别出你。本工具在你的浏览器本地读取这些信号并展示给你；不上传任何数据，扫描全程在你的设备上完成。

什么是浏览器指纹？

浏览器指纹是用你浏览器对外暴露的属性拼出的一份画像：User-Agent 字符串、已装字体、屏幕分辨率、时区、语言列表，以及设备通过 canvas 和 WebGL 渲染图形的方式。由于这些属性的组合常常是独一无二的，即使你屏蔽 cookie 或用无痕模式，网站仍可能在你再次到访时认出你。

canvas 与 WebGL 指纹的原理

当页面把隐藏的文字和图形画到 canvas 再把像素读回时，你的 GPU、显卡驱动和字体渲染上的细微差异会产生一个稳定、且专属于你这台机器的图像哈希。WebGL 指纹则以同样方式读取未被屏蔽的 GPU 厂商与渲染器。这两个信号之所以强，是因为它们能挺过大多数清除 cookie 的操作，并跨会话持续存在。

哪些信号暴露得最多

研究者用「熵的比特数」衡量识别力：比特越多，和你取值相同的人越少。已发表研究把一份完整指纹估为约 18 比特，其中字体、User-Agent、canvas 和 WebGL 渲染器是最强的贡献项。下图展示了主要来源的相对权重。

FAQ

我的数据会被发送到哪里吗？

不会。扫描完全在你的浏览器内用标准 Web API 完成；数值只显示在页面上，绝不上传。

User-Agent 字符串是什么？

它是浏览器随每个请求发送的一行文本，标明浏览器名称、版本和操作系统，是最古老也最基础的指纹信号之一。

canvas 哈希为什么重要？

把图像画到 canvas 再读回，会暴露 GPU 与字体渲染上的细微差异，产生一个常常专属于你设备、且跨会话稳定的哈希。

清除 cookie 能隐藏指纹吗？

不能。指纹来自设备与浏览器属性，而非存储的 cookie，所以它能在你清除 cookie 或使用无痕模式后依然存在。

如何减少我的指纹？

使用会归一信号的浏览器（如 Tor 浏览器或 Brave），保持浏览器更新，并避免使用让你显得与众不同的稀有字体、扩展和屏幕设置。

这些数值准确吗？

它们直接读取自你的浏览器，因此就是任何网站能看到的内容。部分属性（deviceMemory、vendor、platform）已废弃或为近似值，可能缺失。
